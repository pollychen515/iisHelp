Active Server Pages

使用此属性页可以控制 ASP 脚本在所选应用程序内的运行方式。

启用会话状态

使用此复选框可以启用或禁用会话状态。

当启用会话状态时，Active Server Pages 为每个访问 ASP 应用程序的用户创建会话，以便标识访问应用程序中不同网页的用户。当禁用会话状态时，ASP 不跟踪用户，并且不允许 ASP 脚本在“会话”对象中存储信息或者使用“Session_OnStart”或“Session_OnEnd”事件。如果超时期间结束时用户没有请求或刷新应用程序中的网页，会话将自动结束。

要更改超时期间，在“会话超时”文本框中键入新的数字。

通过使用“Session.Abandon”方法，脚本可以显式结束会话。甚至当应用程序的会话状态启用时，也可以通过使用 <%@ ENABLESESSIONSTATE = False %> 命令禁用单个 ASP 网页的会话状态。

启用缓冲

选择此复选框可以缓冲输出到浏览器的信息。

当选中此选项时，ASP 页生成的所有输出在发送到浏览器之前被收集在一起。当清除此选项时，处理页时生成的输出随时返回到浏览器。缓冲输出允许在 ASP 脚本的任何地方设置 HTTP 头。可以通过在脚本中使用“Response.Buffer”方法来覆盖此选项。

启用父路径

选中此复选框可以允许 ASP 脚本使用当前目录的父目录的相对路径（路径使用 .. 语法）。

如果启用此选项，则不要赋予父目录“执行”访问；否则，脚本可能试图在父目录中运行未授权的程序。

默认 ASP 语言

指定 Active Server Pages 的主要脚本语言，此语言用于处理在 ASP 分割符 (<% 和 %>) 之间的进程命令。

要为所选应用程序中的所有页选择其他的主要脚本语言，可在文本框中键入语言名。

ASP 带有两个 ActiveX 脚本引擎：Microsoft Visual Basic Scripting Edition (VBScript) 和 Microsoft JScript (JScript)。默认的 ASP 语言是 vbscript。可以指定任何语言名称，条件是服务器上必须已经安装了该语言的 ActiveX 脚本引擎；必须如脚本引擎提供者的文档所述使用引擎所需的准确关键字。可以通过使用 <%@ LANGUAGE %> 命令来覆盖 ASP 页的默认语言。

脚本超时

指定 ASP 允许脚本执行的时间长度。

如果在超时期间结束时脚本没有运行完毕，ASP 停止脚本并将事件写入 Windows 事件日志。超时期间的值可以设置为在 1 到 2147483647 之间。可以通过在 ASP 脚本中使用“Server.ScriptTimeout”方法来覆盖此选项。

注意：可以在 Web 站点、虚拟目录和目录级别上设置应用程序配置属性。
