主目录属性表

使用此属性表可以更改 Web 站点主目录的位置（默认是 /Wwwroot）。

要更改主目录的位置，选择下列之一：

  • 位于正在配置的服务器上的目录；例如，C:\Catalog\Shoes。要搜索本地驱动器上的文件，请单击“浏览”按钮。
  • 另一计算机上的共享位置；例如，\\webserver\htmlfiles。键入访问该计算机所需的用户名和密码。
  • 重定向到 URL；例如，http://tempserver。将“重定向到”文本框中提供的目标 URL 自动提供给请求原始 URL 上资源的浏览器，以便可以再次请求资源。要将请求映射到虚拟目录，请使用虚拟路径；例如，/newcatalog。

访问权限

读取 如果允许用户读取或下载文件或目录以及相关的属性，请选择该选项。

写入 如果允许用户将 文件及其相关属性上载到您的服务器上或者允许用户更改可写文件内容，请选择该选项。只有使用支持 HTTP 1.1 协议标准的 PUT 特性的浏览器，才能执行写入操作。

脚本源访问 如果要允许用户访问设置了“读取”或“写入”权限的源代码，请选择该选项。源代码包含 ASP 应用程序中的脚本。

目录浏览 要允许用户以超文本形式将虚拟目录中的文件和子目录列表，选择该选项。虚拟目录不会出现在目录列表中，用户必须知道虚拟目录的别名。

要点 如果用户试图访问文件或目录并且下面两个项目为真，则 Web 服务器将在用户 Web 浏览器中显示“Access Forbidden”错误消息。

  • 禁用目录浏览。

  • 用户没有指定文件名，例如 filename.htm。

日志访问 要在日志文件中记录对目录的访问，选择该选项。只有该 Web 站点的日志记录启用之后，才会记录访问。

索引此资源 要允许 Microsoft 索引服务在 Web 站点的完整文字索引中包含这个目录，选择该选项。完整文字索引使用户可在发布的文档中快速搜索单词或短语。

应用程序设置

IIS 基于 Web 的应用程序是通过所在的目录结构来定义的。要使该目录成为应用程序开始点（从而创建一个应用程序），请单击“创建”按钮。要将主目录与应用程序分离，请单击“删除”按钮。

执行权限

执行权限决定允许对该站点或虚拟目录资源进行何种程度的程序执行。

  • 无 只有静态文件（例如 HTML 或图像文件）才可被访问。
  • 仅限于脚本 只有脚本（例如 ASP 脚本）才可被运行。
  • 脚本和可执行程序 所有文件类型都可被访问或者执行。

应用程序保护

选择应用程序是与 Web 服务在同一进程中运行（低）、与其他应用程序一同在分离的共享进程中运行（中），还是在与其他进程分离的单独进程中运行（高）。

客户将被发送到：

如果选择“重定向到 URL”，则出现这些属性。

“上面输入的准确 URL”将虚拟目录重定向到目标 URL，而不添加原始 URL 的任何其他部分。可以使用该选项将整个虚拟目录重定向到一个文件。例如，要将 /scripts 虚拟目录的所有请求重定向到主目录中的文件 Default.htm，请在“重定向到”文本框中键入“/Default.htm”，并选中该选项。

“在这之下的目录”可将父目录重定向到子目录。例如，要将主页目录（由 / 标明）重定向到子目录 /newhome，可在“重定向到”文本框中键入“/newhome”，并选中该选项。如果不使用该选项，Web 服务器会将父目录连续映射到自身。

“此资源的永久重定向”可向客户端发送以下信息：“301 永久重定向”。重定向被视为临时性的，客户端浏览器将接收到以下消息：“302 临时重定向。”某些浏览器可以使用“301 永久重定向”消息作为永久更改 URL的信号（如书签等）。
