侦测未授权的访问

可以查看 IIS 日志记录和 Windows 安全日志以监视扩展时间周期上的安全事件。可以使用 Microsoft 管理控制台查看 Windows 安全日志。IIS 日志可以用任何文本编辑器或文字处理软件来查看。关于查看 IIS 日志的详细信息，请参阅记录站点活动。

在 Windows 安全日志中，您可以发现未授权的访问尝试，它们显示为警告或错误日志条目。还可以将这些日志存档以备后用。关于审核的详细信息，请参阅 Windows 文档。

查看 Windows 安全日志检测潜在的安全性问题
  1. 单击“开始”，指向“设置”，单击“控制面板”，双击“管理工具”，然后双击“计算机管理”。
  2. 展开“系统工具”。
  3. 展开“事件查看器”。
  4. 选择“安全日志”。

    5. 注意 如果无法查看安全日志，则说明您所使用的用户帐户不具有这一权限。这是由于域级安全策略覆盖了本地计算机级的安全策略，也就是说您可以作为本地计算机的管理员登录，但不能访问计算机的安全日志。要获得许可权限，请与网络管理员联系。关于安全策略的详细信息，请参阅计算机文档。

  5. 检查可疑安全事件的日志文件，包括下列各项：
    • 无效登录尝试。
    • 权限使用失败。
    • 试图访问并修改 .bat 或 .cmd 文件失败。
    • 尝试改变安全权限或审核日志。
    • 尝试关闭服务器。
将 Windows 安全日志存档
  1. 单击“开始”，指向“设置”，单击“控制面板”，双击“管理工具”，然后双击“计算机管理”。
  2. 展开“系统工具”。
  3. 展开“事件查看器”。
  4. 选择“安全”。
  5. 在“动作”菜单上，单击“日志文件另存为”。
  6. 在“另存为”对话框中，选择想要保存文件的目录，然后键入该文件的名称。

    7. 注意 安全日志可以保存为事件 (.evt) 文件、文本 (.txt) 文件或逗号分割的 (.csv) 文件。

打开存档的 Windows 安全日志
  1. 单击“开始”，指向“设置”，单击“控制面板”，双击“管理工具”，然后双击“计算机管理”。
  2. 展开“系统工具”。
  3. 展开“事件查看器”。
  4. 在“日志”菜单上，选择“安全”。
  5. 在“动作”菜单上，指向“新建”，并单击“日志查看”。
  6. 在“添加另一个日志查看”对话框中，选择“保存（打开一个先前保存的日志文件）”并浏览文件。
  7. 在“日志类型”下拉列表中，选择“安全”。
  8. 单击“确定”在查看器中打开该文件。
查看 IIS 日志文件检测可能的安全性问题
  1. 在文本编辑器（如记事本）中打开日志文件。关于日志文件的详细信息，请参阅记录站点活动。
  2. 检查可疑安全事件的日志文件，包括下列各项：
    • 多个尝试运行可执行文件或脚本的无效的命令。（必须严密监视脚本目录。）
    • 单个 IP 地址的无效登录尝试过多，目的可能是增加网络通信量或拒绝访问其他用户。
    • 试图访问并修改 .bat 或 .cmd 文件失败。
    • 将文件上载到包含可执行文件目录的未授权尝试。
