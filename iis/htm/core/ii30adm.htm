向下兼容式站点管理

您可以使用 IIS 4.0 附带的 IIS 管理单元远程管理 Intranet 中的 IIS 5.0。要通过 Internet 或代理服务器远程管理 IIS，应使用基于浏览器的 Internet 服务管理器 (HTML)。

另外，您可以指派一个 Web 站点，通过使用 IIS 3.0 管理接口的应用程序对该站点进行程序化管理。

注意 在 IIS 3.0 中，对于没有与特定 IP 地址关联的任何虚拟目录，都可以从该计算机上的“全部”Web 站点访问。该行为在 IIS 4.0 和 5.0 中有了变化，要从使用多个 IP 地址的多个 Web 站点访问虚拟目录，必须将该虚拟目录添加到“各个”站点中。

为 IIS 3.0 管理指定可用的站点
  1. 在 Internet 信息服务管理管理单元中，选择计算机图标并打开其属性页。
  2. 选择“WWW 服务”或“FTP 服务”并单击“编辑”按钮。
  3. 单击“服务”选项卡。
  4. 在“IIS 3.0 管理”中，选择站点，然后单击“确定”。

要点 “应用程序映射”属性页的“动作”列中列出了将由应用程序运行的 HTTP 动作。在 IIS 4.0 中，“排除”列内列出了不能由应用程序运行的 HTTP 动作。如果在运行 IIS 4.0 和 IIS 5.0 的计算机间执行远程管理任务，该列标记将出现错误。

关于应用程序映射的详细信息，请参阅设置应用程序映射。
