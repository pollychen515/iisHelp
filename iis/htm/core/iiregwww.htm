WWW 服务注册表项目

除了在服务特有的注册表项目中列出的参数外，WWW 服务还保留了下列参数。

注册表路径：

HKEY_LOCAL_MACHINE\SYSTEM
\CurrentControlSet
 \Services
  \W3SVC
   \Parameters

AcceptByteRanges
REG_DWORD
范围： 1、0
默认：1（启用）
该值决定 HTTP 服务器是否将为“字节：”类型处理“范围”头部。如果启用，服务器将发出信号表示它接受范围请求，通过发送“接受范围：字节”头字段，并处理进来的请求，根据 Internet 草案“字节范围扩充到 HTTP”，指定“范围：字节=”头字段。

 

AllowSpecialCharsInShell
REG_DWORD
范围： 0、1
默认：0（禁用）
该值控制当运行批处理文件（.bat 和 .cmd 文件）时，命令行是否允许 Cmd.exe 特殊字符（包括 | ( , ; % < 和 >)。这些特殊字符可能引起严重的安全问题。如果该项目的值设置为 1，怀有恶意的用户可以执行服务器上的命令。所以，强烈建议将它设置为 0，即默认值。默认情况下，这些特殊字符不能传递给脚本映射 CGI 小程序。如果设置为 1，这些特殊字符可以传递给脚本映射 CGI 小程序，除了管道字符 (|) 和标准 I/O 重定向字符(< 和 >)，它们对于命令处理器有特殊的含义。

 

DLCSupport
REG_DWORD
范围： 0、1
默认：0（禁用）
该值指出是否将启用低级客户支持。一些早期版本的浏览器不支持主机头，主机头给予客户访问共享一个 IP 地址的多个 Web 站点的能力。值为 0 意味着将不支持低级客户。值为 1 意味着将支持低级客户。

 

DLCCookieNameString
REG_STRING
范围： 字符串
默认：不适用
该值指定服务器发送给低级客户的 HTTP cookie 的名称。发送给低级客户的 cookie 将行使伪主机头的功能，这样服务器就可以将客户的 HTTP 请求发送到适当的 Web 站点。

 

DLCHostNameString
REG_STRING
范围： 字符串
默认：不适用
该值识别包含低级主机菜单的 Web 站点的名称，该菜单保存在 LCCookieMenuDocumentString。低级主机菜单是一个文件（如 HTML 文件、asp 文件等等），它列出了单个 IP 地址上的所有 Web 站点。用户从该菜单选择适当的服务器实例。

 

DLCCookieMenuDocumentString
REG_STRING
范围： 字符串
默认：不适用
该值为支持 cookies 但不支持主机头的客户指定主机菜单的文件名。Cookie 作为“伪”主机头，而主机菜单列出共享单个 IP 地址的所有 Web 站点。Cookie 附加于所有对主机的请求，直到浏览器会话被终止；但是，终止浏览器会话并不删除 cookie。

 

DLCMungeMenuDocumentString
REG_STRING
范围： 字符串
默认：不适用
该值为不支持 cookies 的客户指定主机菜单的文件名。这个文件用来从低级客户解析（或 “munge”）请求的 URL，方法是在 URL 中嵌入主机名。

 

DLCMenuString
REG_STRING
范围： 字符串
默认：不适用
该值指定低级客户请求的 URL 的特殊前缀。服务器将检查所有对该字符串的低级请求。

 

LogSuccessfulRequests
REG_DWORD
范围： 0、1
默认：1（启用）
确定是否在日志文件中记录成功的活动。值为 1 时记录成功的活动，为 0 则将它关闭。

 

SSIEnableCmdDirective
REG_DWORD
范围： 0, 1
默认：0（禁用）
为了执行 shell 命令，使用了服务器端包含的 #exec cmd 指令。具有安全意识的站点可能希望通过将该值设置为 0 来禁用 #exec cmd 命令，以作为附加的安全防范，特别是当允许不受信任的团体将文件放置在服务器上时。默认情况下该值不存在于注册表中；要允许该命令执行外壳命令，必须首先创建该值并将其设置为 1。

 

TryExceptDisable
REG_DWORD
范围： 0、1
默认：0（禁用）
该参数将在调用 ISAPI 应用程序的 HttpExtensionproc() 时禁用异常高速缓存。将 TryExceptDisable 设置为 1 将导致奇怪的 ISAPI 应用程序引起的异常停止服务器，但它允许开发者进行及时的调试；因此仅在调试 ISAPI 应用程序时才将该参数设置为 1。

 

UploadReadAhead
REG_DWORD
范围： 0 - 0x80000000
默认：48KB
当客户把数据发送到服务器时，这将是服务器把控制传递给应用程序之前所读取的默认数量。应用程序将负责读取其余的数据。如果要增加大小，则应增加服务器 RAM 的数量。

 

UsePoolThreadForCGI
REG_DWORD
范围： 1、0
默认：1（启用）
默认情况下，IIS 使用服务器池线程进行 CGI 处理。这意味着扩展期间的 CGI 请求将消耗一个服务器池线程。
