设置服务器上的 SSL

可以配置 Web 服务器安全套接字层 (SSL) 的安全功能以检验内容完整性、用户标识和加密网络传输。

在 Web 服务器上设置 SSL
  1. Web 服务器要求有效的服务器证书以建立 SSL 通信。使用 Web 服务器证书向导生成发送到证书颁发机构的证书请求文件（默认情况下为 NewKeyRq.txt），或生成对联机证书颁发机构（如 Microsoft 证书服务）的请求。详细信息，请参阅使用新的安全任务向导。

    如果未使用 Microsoft® Certificate Services 2.0 颁发自身的服务器证书，则必须由第三方的证书颁发机构批准您的请求并颁发您的服务器证书。详细信息，请参阅获取服务器证书。

    2. 注意 根据服务器证书提供的标识保证级别，您可能需要等待几天到几个月，以便证书颁发机构批准您的请求和发送证书文件。

  2. 接收到服务器证书文件后，可以使用向导安装它。安装进程将证书附加或“绑定”到 Web 站点。

    3. 注意 每个 Web 站点只能有一个服务器证书。

  3. 在 Internet 信息服务管理单元中，选择需使用 SSL 保护的 Web 站点并打开其属性页。在“Web 站点”属性页的“Web 站点标识”下，选择“高级”。
  4. 在“高级多 Web 站点配置”对话框的“此 Web 站点的多个 SSL 标识”下，确保 Web 站点 IP 地址分配到端口 443，即安全通信的默认端口。

    每个 Web 站点可以有多个 SSL 端口。要配置更多的 SSL 端口，单击“此 Web 站点的多个 SSL 身份”下的“添加”。

  5. 在“目录安全”或“文件安全”属性页的“安全通信”下，单击“编辑”。
  6. 在“安全通信”对话框中，将 Web 服务器配置为需要安全频道。如果要求 128 位的密钥加密，请确保用户的 Web 浏览器支持 128 位加密。详细信息，请参阅加密。

    7. 注意 由于出口限制，128 位密钥长度加密功能仅可以在美国和加拿大使用。有关升级到 128 位加密功能的信息，请访问 Windows 2000 Server 的支持 Web 站点 http://support.microsoft.com/support。

  7. 在安全通信下，单击“编辑”。可以选择启用 Web 服务器的 SSL 客户证书验证和映射功能。请参阅以下内容：
    • 启用客户证书
    • 将客户证书映射到用户帐户
