安全通信

使用该对话框为使用支持安全通信（URL 以 https:// 开头）的 Web 浏览器的来访者配置安全套接字层 (SSL) 加密通信。

在可以使用 Web 服务器的 SSL 安全功能之前，必须从一个可信任的、称为“证书颁发机构”的第三方机构获得一个有效的服务器证书。这不能在 Internet 服务管理器 (HTML) 中实现；必须使用 IIS 管理单元。

需要安全通道 (SSL) 要求 Web 浏览器与 Web 站点、目录或文件的连接为加密通信连接，请选择该选项。

需要 128 位加密 要求 Web 浏览器与 Web 站点、目录或文件的连接为 128 位加密通信连接，请选择该选项。

要点 由于受到出口限制，128 位密钥长度加密功能只能在美国和加拿大使用。

客户证书

可以配置 Web 服务器为接受、要求或拒绝客户证书，作为建立与特定文件或目录的连接的方法。客户证书是一个可信任的、第三方的机构颁发的数字标识，该机构称为 “证书颁发机构”。

不接受客户证书 要允许用户访问而不要求出示客户证书，请选择该选项。

接受客户证书 要允许有客户证书的用户访问，但并不要求证书，请选择该选项。有客户证书的用户可用于映射；没有证书的用户可使用其他验证方法。

需要客户证书 为了只允许使用有效客户证书的用户连接，请选择该选项。将拒绝没有有效客户证书的用户的访问。

需要安全通道 (SSL) 要使用该选项，必须选择复选框。

启用客户证书映射

要验证使用有效客户证书登录的用户，请选择该选项。映射将包含在客户证书中的信息与 Windows 用户帐户信息相匹配。
