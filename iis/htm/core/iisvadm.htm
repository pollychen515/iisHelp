服务器管理

欢迎访问关于服务器管理的主题。本节描述了如何创建附加的 Web 和 FTP 站点以及更改默认设置。还包括如何记录站点通信和监视服务器性能。

可以在 IIS 中使用默认设置，也可以自定义这些设置以适应自己 Web 发布的需要。通过调整默认设置，可以提供附加值、更好的性能以及更高的安全性。

本节包括：

  • Web 和 FTP 站点：关于 Web 和 FTP 站点的基本信息，包括操作指导、远程管理和命名站点。
  • 名称解析：设置 WINS 或 DNS 名称解析系统，以便将“友好”名称正确映射到 IP 地址。也包括关于通过分配端口、地址和主机头名维护多个站点的概念信息。
  • 配置应用程序：控制应用程序作用域以获得最佳的可靠性和性能。
  • 安全：保护服务器、内容和应用程序不受非法的入侵。
  • 自定义错误消息：为用户提供对用户友好的自定义 HTTP 1.1 错误消息。
  • 记录站点活动：跟踪站点通信和解释日志文件。
  • 性能调整：监视和提高服务器性能。
  • 复制和群集：为 IIS 提供容错和负载平衡。
  • 管理脚本：使用命令行脚本管理服务器。

本节不包括：

  • 虚拟目录；请参阅 Web 站点管理。
  • 脚本；请参阅 Active Server Pages。
