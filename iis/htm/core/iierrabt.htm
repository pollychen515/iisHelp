关于自定义错误消息

该主题包括有关默认和自定义 HTTP 1.1 错误的总体信息。

  • HTTP 1.1 错误消息
  • 自定义错误消息

HTTP 1.1 错误消息

如果用户试图连接到 Web 站点时出现了 HTTP 错误，将向用户返回一条一般性消息，简述试图建立连接时所发生的事情。例如，如果用户试图连接某个 Web 站点而该站点已经达到最大连接数，此时将以 HTML 页形式返回一条 HTTP 错误，其中包含“用户太多”语句。

自定义错误消息

可以使用 IIS 中的“自定义错误”属性页来自定义一般的 HTTP 错误。

注意 所有 IIS 自定义错误消息都显示行业标准 HTTP 代码，这样可以确保与 HTTP 1.1 错误消息一致。

注释

  • 自定义错误消息作为列表显示在 IIS 的管理单元中。IIS 将此管理单元看作单个属性。例如，在 Web 站点级配置一组自定义错误消息之后，该服务器下的所有目录都将继承整个自定义错误消息列表。也就是说，两个自定义错误消息列表（对于服务器和目录）并不合而为一。
  • 如果 Web 服务器超过了 CPU 进程限制，就会向客户端发送一条 HTTP 错误 503.1 的自定义错误。这条错误消息向客户端传递这样的信息，即由于 Web 服务器超出这些设置限制，已被暂停或停止。详细信息，请参阅限制进程。
  • 404.1 错误只会出现在具有多个 IP 地址的计算机上。如果在特定 IP 地址/端口组合上收到客户请求，而且在特定的端口上 IP 地址并没有设置为侦听，则 IIS 将返回 404.1 HTTP 错误。例如，如果一台计算机有两个 IP 地址，而只将其中一个 IP 地址配置为在端口 80 上侦听，则其它 IP 地址从端口 80 收到的任何请求都将导致 IIS 返回 404.1 错误。只应在服务级设置这一错误，因为只有当服务器上使用多个 IP 地址时它才返回给客户。
