控制数据库访问

可以使用各种技术以有效地限制对数据库文件的访问。这些技术的范围包括从配置数据库软件的安全功能到设置数据库文件的 NTFS 权限等。本主题概述有效地限制对 Web 服务器数据库文件访问的基本准则。

控制数据库访问
  1. 在创建数据库的同时配置数据库软件的安全功能。使用某些版本的数据库软件提供的安全功能，可以控制用户使用较高等级访问数据库文件。例如，使用 Microsoft SQL Server 的安全功能，可以控制用户如何访问特定数据库文件，包括这些用户如何访问特定的表、记录和字段。详细信息，请参阅数据库软件文档。
  2. 通过适当设置 NTFS 权限，可保护存储在 Web 服务器上的数据库文件和目录的安全。确保需要更新数据库文件的受信用户具有适当的权限。详细信息，请参阅设置目录或文件的 NTFS 权限。
  3. 使用适当的 NTFS 权限可以保护文件的数据源名 (DSN)，该数据源名可能包含用户 ID 和密码。
  4. 使用适当的 NTFS 权限可保护 .asp 文件，该文件包含将用户名和密码信息传递给数据库的脚本。

    5. 提示 请考虑将 ASP 数据库脚本（特别是那些包含用户名和密码信息的脚本）转换成安全的 COM 服务器组件。详细信息，请参阅随 Platform SDK 发布的 Internet 信息服务软件开发工具包。

  5. 选择适当的 Web 服务器验证方法识别尝试访问数据库的用户身份。数据库用户验证在很大程度上可以依赖于用来连接数据库的协议。例如，如果决定使用 SQL Server 的命名管道默认连接协议，则对 Windows 用户帐户凭据的验证可能会发生在另外的 SQL Server 验证中。应仔细参考数据库软件文档，以获得关于集成不同验证方法的规则。
