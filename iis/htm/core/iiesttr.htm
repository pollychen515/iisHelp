估计通信量

本主题解释了如何测算和设置站点每天的预期连接数量。如果该数量只是比实际连接的数量稍微高一点，连接将变得更快，而且服务器性能得到改善。如果该数量远远大于实际的连接尝试数量，既浪费了服务器内存，又降低服务器的总体性能。

要确定服务器每天的连接数量
  1. 使用系统监视器记录“总计连接请求数”计数器和“当前连接数”计数器的值，它们都在 Web 服务对象中。
  2. 如果可能的话，收集几天的日志文件数据，以获得服务器运行状况的现实记录。

    3. 注意 一些连接请求可能失败，所以“总计连接请求数”计数器不一定十分精确。

要设置估计每天的连接数量
  1. 在 Internet 信息服务管理单元中， 选择 Web 站点，并单击“属性”按钮显示属性表。
  2. 在“性能”属性表中，设置一个比在 24 小时期间内接收的连接数量估计值稍大的数值。
