高速缓存应用程序

通过调整与在服务器上运行的 Web 应用程序相关的几个设置，可以调整服务器的性能。记录使用情况数据，如请求数量和花费的时间，将有助于决定如何进行调整。关于服务器活动日志的详细信息，请参阅记录 Web 站点活动。关于性能问题的附加信息，请参阅关于性能调整。

包括下面的设置：

  • 设置 CGI 脚本超时
  • 高速缓存 ISAPI 扩展
  • 高速缓存 ASP 脚本文件

设置 CGI 脚本超时

除了与连接相关的超时外，还可以设置 CGI 脚本的超时值。此超时值是 CGI 脚本执行的时间数量。CGI 应用程序继承了整个 Web 站点的超时设置。可以更改整个 Web 站点或者单独应用程序的设置。

要设置超时值
  1. 在 Internet 信息服务管理单元中，选择计算机图标并打开其属性表。
  2. 在“主属性”下，选择“WWW 服务”，然后单击“编辑”按钮。
  3. 单击“主目录”选项卡。
  4. 单击“配置”按钮。
  5. 单击“进程选项”选项卡，然后在“CGI 脚本超时”框中键入超时值。

高速缓存 ISAPI 扩展

可以加载和缓存 ISAPI DLL，以便处理进一步的请求时不用再次调用它们。否则，DLL 将在运行之后被卸载。缓存减少了用户等待加载 DLL 的时间。大多数 ISAPI 扩展（包括 Active Server Pages）都受益于缓存。只有在特定环境下，如调试 ISAPI 扩展，才可以清除该选项。

要启用高速缓存
  1. 在 Internet 信息服务管理单元中，选择 Web 站点或应用程序的起始点目录。
  2. 打开目录的属性表，然后单击“主目录”、“虚拟目录”或“目录”选项卡。
  3. 单击“配置”按钮。
  4. 单击应用程序映射选项卡，然后选择“高速缓存 ISAPI 扩展”复选框。

高速缓存 ASP 脚本文件

ASP 处理包含 ASP 脚本的文件，并在内存中缓存此文件以便提高性能。通过增加缓存文件数，可以进一步提高性能。可以为在 Web 站点上运行的所有应用程序设置高速缓存文件数，也可以更改单个应用程序的设置。

  1. 在 Internet 信息服务管理单元中，选择 Web 站点或应用程序的起始目录。
  2. 打开目录的属性表，然后单击“主目录”、“虚拟目录”或“目录”选项卡。
  3. 单击“配置”按钮，然后单击“进程选项”选项卡。
  4. 要高速缓存所有请求的 .asp 文件，选择“所有请求的 ASP 文件放入高速缓存” 选项。
  5. 要设置高速缓存的脚本数，选择“放入高速缓存的最多 ASP 文件数”选项并在文本框中键入数值。
