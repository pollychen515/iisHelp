Web 和 FTP 站点

起初，每个域名（如 www.microsoft.com）表示一台单独的计算机。目前，运行 Windows 2000 Server 的单个计算机可维护多个 Web 或 FTP 站点，但表面上看来是几台计算机。每个 Web 站点都具有维护一个或多个域名的能力。由于每个站点都模仿单个计算机的外观，因此站点有时被认为是“虚拟服务器”。

本节包括：

  • 关于 Web 和 FTP 站点：描述 Web 和 FTP 站点及其属性的概念信息。
  • 启动和停止站点：关于使用 IIS 管理单元来启动和停止站点的过程信息。
  • 添加站点：在运行 Windows 2000 Server 的计算机上维护多个站点。
  • 设置 FTP 消息和目录输出样式：为 FTP 客户创建自定义消息和目录列表格式。
  • 命名 Web 站点：将主机头名、IP 地址和端口的唯一组合分配给 Web 站点。
  • 重新启动 IIS：通过使用 IIS 管理单元来停止和启动 Internet 服务。
  • 支持旧浏览器中的主机头名：解析旧浏览器中 URL 的过程，这些浏览器不支持使用 Web 站点主机头名。
  • 更改继承的默认值：更改站点的全局默认设置。
  • 备份和还原 IIS:保存和重新应用 IIS 配置设置。
  • 分配 Web 站点操作员：允许指定的 Windows 用户帐户对特定的 Web 站点具有有限的管理特权。
  • 远程管理：通过使用 Internet 信息服务管理单元或基于浏览器的 Internet 服务管理器 (HTML) 远程管理站点。
  • 下层站点管理：使用旧版本的 Internet 服务管理器远程管理 IIS 5.0 安装的一个 Web 站点和一个 FTP 站点。

本节不包括：

  • 网络命名系统和维护多个站点；请参阅名称解析。
  • 增强安全设置；请参阅安全。
  • 添加虚拟目录；请参阅创建虚拟目录。
