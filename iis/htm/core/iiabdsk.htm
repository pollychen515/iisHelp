重定向参考

重定向客户的请求是确保用户得到正确页面的一种方法，它适用于站点正在建设之中或已改变标识的情况。可以将用户请求重定向到运行 IIS 的服务器的目录、另一计算机上的 UNC 共享位置或另一个 URL。

重定向变量

可以使用重定向变量将原始 URL 中相关的部分传递给目标 URL。通过打开 IIS 管理单元中目录的属性页，然后单击“主目录”、“虚拟目录”或“目录”选项卡，可以使用这些变量。

变量 说明 示例
$S 传送与所请求 URL 的相匹配的后缀。匹配后缀是用重定向的 URL 代替之后，原始 URL 中所保留的部分。 如果 /scripts 重定向到 /newscripts 而原来的请求针对/scripts/program.exe，那么 /program.exe 便是后缀。服务器自动执行后缀替换，$S 变量仅在与其他变量结合时使用。
$P 传送原来 URL 中的参数。 例如，如果原来的 URL 是 /scripts/myscript.asp?number=1，那么字符串“number=1”映射到目标 URL。
$Q 传送原始 URL 的问题标志和参数。 例如，如果原来的 URL 是 /scripts/myscript.asp?number=1，那么字符串“?number=1”映射到目标 URL。
$V 传送请求的 URL，不包括服务器名。 例如，如果原来的 URL 是 //myserver/scripts/myscript.asp，那么字符串 “/scripts/myscript.asp” 将映射到目标 URL。
$0 到 $9 传送被请求的 URL 中与指出的通配符相匹配的部分。  
! 不重定向。 使用此变量可防止对已重定向的虚拟目录中的子目录或单个文件进行重定向。

重定向通配符

可以使用重定向通配符匹配原来 URL 中的任意数目的字符。在“重定向到”文本框中插入通配符 (*)，文本框位于目录的“主目录”、“虚拟目录”或“目录”选项卡。目标 URL 可以星号和分号开头，使用分号可将成对的通配符和目标 URL 分开。

例如，要将所有对 /scripts/filename.stm 的请求重定向到名为 Default.stm 的文件，将所有对 /scripts/filename.htm 的请求重定向到 Default.htm 文件，对于 /scripts 虚拟目录，在“重定向到”文本框键入：

*;*.stm;/default.stm;*.htm;/default.htm

当使用通配符时，确保选择了“上面输入的准确 URL”复选框。对于重定向路径，在 URL 中键入。
