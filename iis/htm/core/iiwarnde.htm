创建应用程序

要创建应用程序，应将目录指定为应用程序的起点（应用程序根）。而后可以为应用程序设置属性。每个应用程序都可以有一个好记的名称；该名称出现在 Internet 信息服务管理单元中并给定一种区分应用程序的方法。不在其他任何地方使用该应用程序名。

可以从应用程序边界删除目录。对该目录及其子目录中文件的请求将不再启动该应用程序。从应用程序边界中删除目录并不会从 Web 站点或计算机硬盘上删除该目录。

创建应用程序
  1. 在 Internet 信息服务管理单元中，选择作为应用程序起点的目录。可以将 Web 站点的主目录指定为应用程序起点。
  2. 打开目录的属性表，然后单击“主目录”、“虚拟目录”或“目录”选项卡。
  3. 单击“创建”按钮。
  4. 在“应用程序名”文本框中，键入应用程序名。

    5. 注意 如果是一个“删除”按钮而不是“创建”按钮，则已经创建了一个应用程序。

 

从应用程序中删除目录
  1. 在 Internet 信息服务管理单元中，选择要删除的目录。
  2. 打开目录的属性表，然后单击“主目录”、“虚拟目录”或“目录”选项卡。
  3. 单击“删除”按钮。

技巧

  • 要终止应用程序并将其从内存中卸载，请单击“卸载”按钮。如果“上载”按钮无效，则表明没有位于应用程序的起始点目录。
  • 要将主目录与应用程序分离，请单击“删除”按钮。
  • 选择“在单独的内存空间运行（独立进程）”复选框，在独立于 Web 服务器进程的进程中运行应用程序。运行隔离的应用程序可以使其他应用程序（包括 Web服务器自身）在该应用程序失败或终止响应时免受影响。
为应用程序设置权限
  • 将权限设为“无”以防止任何程序或脚本运行。
    或
  • 设置为“纯脚本”以便在无须设置“执行”权限的情况下启用映射到脚本引擎的应用程序在该目录中运行。使用包含 ASP 脚本、Internet 数据库连接器 (IDC) 脚本或其他脚本的目录的“脚本”权限。“脚本”权限比“执行”权限更安全，因为可以限制在该目录中运行的应用程序。有关将应用程序变为脚本引擎的信息，请参阅应用程序映射属性页。
    或
  • 设置“脚本和可执行程序”以便允许任何应用程序在该目录中运行，这包括映射到脚本引擎和 Windows 二进制文件（.dll 和 .exe 文件）的应用程序。
