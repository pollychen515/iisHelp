扩充日志记录属性

通过选择记录在日志中的字段，可以自定义 W3C 扩充日志。要限制日志大小，可忽略不需要的域。

字段包括：

  • 日期 活动发生的日期。
  • 时间 活动发生的时间。

扩充属性

  • 客户 IP 地址 访问服务器的客户 IP 地址。
  • 用户名 访问服务器的用户名。
  • 服务名 在客户计算机上运行的 Internet 服务的名称。
  • 服务器名 生成日志条目的服务器的名称。
  • 服务器 IP 生成日志条目的服务器的 IP 地址。
  • 服务器端口 客户连接的端口号。
  • 方法 客户试图执行的活动（如，GET 命令）。
  • URI 资源 访问的资源：例如，HTML 页、CGI 程序或脚本。
  • URI 查询 客户试图执行的查询；即，客户为其寻找匹配项目的一个或多个搜索字符串。
  • HTTP 状态 用 HTTP 术语表示的活动状态。
  • Win32 状态 用 Windows 术语表示的活动状态。
  • 发送的字节数 服务器发送的字节数量。
  • 接收字节数 服务器接收的字节数量。
  • 所花时间 活动花费的时间长度。
  • 协议版本 客户使用协议（HTTP、FTP）版本。对于 HTTP，这将是 HTTP 1.0 或 HTTP 1.1。
  • 用户代理 客户使用的浏览器。
  • Cookie 发送或接收的 cookie 内容，如果存在。
  • 参照 一种站点，用户单击其上面的链接就将用户带入该站点。

CPU 帐户

  • 事件 触发事件的进程类型：CGI 或进程外的应用程序。类型可以是 CGI、应用程序或全部。
  • 进程类型 触发的事件：Site-Stop、Site-Start、Site-Pause、Periodic-Log、Interval-Start、Interval-End、Interval-Change、Update、Eventlog-Limit、Priority-Limit、Process-Stop-Limit、Site-Pause-Limit、Eventlog-Limit-Reset、Priority-Limit-Reset、Process-Stop-Limit-Reset 或 Site-Pause-Limit。
    • Site-Stop 停止 Web 站点。
    • Site-Start 启动或重启动 Web 站点。
    • Site-Pause 暂停 Web 站点。
    • Periodic-Log 这是一个有规律地定义的日志条目，其间隔由管理员指定。
    • 重置 Interval-Start 开始“重置间隔”。
    • 重置 Interval-End 到达“重置间隔”并重置。
    • 重置 Interval-Change Web 站点管理员更改“重置间隔”的值。
    • Update 发生了下列某个事件：更改日志间隔、发生间隔事件、站点停止、启动或暂停。
    • Eventlog-Limit 因为 Web 站点的 CGI 和外部进程应用程序所用 CPU 资源达到了管理员设置的事件日志限制，所以生成该站点的事件日志。
    • Priority-Limit 对 Web 站点中达到管理员设置的低优先级限制的 CGI 或外部进程应用程序设置低优先级。
    • Process-Stop-Limit 停止 Web 站点的某个 CGI 或外部进程应用程序，因为它达到了管理员设置的进程停止限制。
    • Site-Pause-Limit 暂停 Web 站点，因为它的某个 CGI 或外部进程应用程序达到了管理员设置的站点暂停限制。
    • Eventlog-Limit-Reset 达到“重置间隔”或者手工更改了 Eventlog-Limit。
    • Priority-Limit-Reset 达到“重置间隔”或者手工更改了 Priority-Limit。
    • Process-Stop-Limit-Reset 达到“重置间隔”或者手工更改了 Process-Stop-Limit。
    • Site-Pause-Limit 达到“重置间隔”或者手工更改了 Site-Pause-Limit。
  • 用户时间 站点在当前间隔使用的总计用户模式处理器时间，以秒为单位。
  • 内核时间 站点在当前间隔使用的总计内核模式处理器时间，以秒为单位。
  • 页面错误 导致内存页面错误的内存参照总数。
  • 进程总计 在当前间隔内创建的 CGI 或外部进程应用程序的总数。
  • 活动进程 当记录日志时正在运行的 CGI 或外部进程应用程序的总数。
  • 终止进程 在当前间隔中由于进程限制而停止的 CGI 和进程外应用程序的总数。
