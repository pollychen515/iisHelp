限制连接数和设置连接超时

限制连接是保留带宽用于其他用途的一种方法，这些用途包括电子邮件、新闻服务器或运行于同一安装位置的其他 Web 站点等。连接限制之外的所有连接尝试都被拒绝。如果浏览器意外停止工作或连接中途失效，站点将继续处理数据，直到达到超时值。设置超时值可减少由于长时间处理断开的连接而造成的处理资源的损失。

确定是否限制连接
  1. 至少在 Web 服务和 FTP 服务对象上使用“系统监视器”记录当前连接数、最大连接数和连接尝试总数计数器。
    2. 继续记录一段时间，直到完全把握常规范围。通常，这可能需要几天到一周，甚至更长的时间。
限制连接数量
  1. 在 Internet 信息服务管理单元 中， 选择 Web 站点，并单击“属性”按钮显示其属性页。
  2. 在“Web 站点”属性页中，选中“限制到”选项。
  3. 在“最多连接数”框中键入允许同时连接的最大数目。

    4. 注意 “无限”选项允许同时连接的数量就是带宽和处理器所能支持的最大数量。

  4. 在“连接超时”框中，键入以秒为单位的连接超时值。
