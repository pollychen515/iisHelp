Web 站点属性表

使用此属性表可以设置 Web 站点的标识参数。对于即将在此属性表上可用的站点 IP 地址，必须首先在被管理的计算机上配置站点的 TCP/IP 设置。

标识

说明

键入服务器名称。此名称显示在 Internet 服务管理器 (HTML) 的树视图中。

单击“高级”按钮可以配置 IP 地址、TCP 端口号、SSL 端口号以及主机头名。

IP 地址

对于显示在此框中的地址，它必须已经在“控制面板”中定义为在被管理的计算机上使用。详细信息，请参阅 Windows 文档。如果没有分配指定的 IP 地址，此站点将对应于分配给此计算机但没有分配给其它站点的所有 IP 地址，这使它成为默认 Web 站点。

TCP端口

确定服务正在运行的端口。默认值是端口 80。可以将端口更改为任何唯一的 TCP 端口号；客户端必须事先知道才能请求该端口号，否则请求不能连接到服务器。端口号是必需的，不能为空。

连接

无限

选择此选项可以不限制同时发生的连接数。

限制为

选择此选项可以限制同时连接到站点的最大连接数。在此对话框中，键入允许的连接数最大值。

连接超时

以秒为单位设置服务器断开不活动用户之前保持连接的时间长度。这将确保在 HTTP 协议无法关闭某个连接时，关闭所有的连接。

启用 HTTP 保持活动

允许客户与服务器保持开放连接，而不是根据每个新请求重新打开客户连接。禁用“保持活动”可能降低服务器性能。默认情况下启用“保持活动”。

启用日志

选择此选项可以启用 Web 站点的日志功能，它可以记录关于用户活动的细节并按所选格式创建日志。启用日志后，请从“活动日志格式”列表中选择格式。格式如下：

  • Microsoft IIS 日志文件格式 - 一种固定的 ASCII 格式。
  • NCSA 常用日志文件格式 - “超级计算应用程序国家中心 (NCSA)”的常用格式；一种固定的 ASCII 格式。
  • W3C 扩展日志文件格式 - 一种可自定义的 ASCII 格式，默认选择此格式。为了使用进程帐户，必须选择此格式。
  • ODBC 日志（只适用于 Windows 2000 Server）- 一种固定的数据库日志格式。

要配置日志文件创建（例如，每周或按文件大小）选项，或者配置 W3C 扩展日志或 ODBC 日志的属性，请单击“属性”。
