  1. 如何发布文档或 Web 页？

    2. 要发布文档，请将文件移到 \inetpub\wwwroot 目录。接着再在浏览器的地址栏中键入 http://服务器名/文件名，查看已发布的文件。详细信息，请参阅使用 IIS 快速进行站点设置。

  2. 如何在 Web 或 FTP 站点上创建虚拟目录？

    3. 可以使用 Internet 信息服务管理单元在 Web 站点上创建虚拟目录。主题创建虚拟目录包含关于如何完成此任务的过程信息。

  3. 如何创建 Web 站点？

    4. 在 Microsoft® Windows 2000 上安装 IIS 时，已经为您设置了一个默认的 Web 站点。可以立即在默认发布地点发布您的内容。详细信息，请参阅添加站点。

  4. 如何创建多个 Web 站点？

    5. 要创建多个 Web 站点，必须首先确保每个站点都有唯一的标识。为此，需要与网络管理员联系，获得多个 IP 地址或者分配多个主机头名。详细信息，请参阅关于名称解析。

  5. 如何复制 Web 内容？

    6. 可以使用“内容部署”服务（以前称为“内容复制系统”，即 CRS）在 Web 群集的计算机之间复制内容和设置。“内容部署” Microsoft® 站点服务器的一个特征。关于“内容部署”服务的详细信息，请参阅 Site Server 文档。关于群集的详细信息，请参阅复制和群集。

  6. 如何停止和重新启动 Internet 服务？

    7. 几个 Internet 服务可能在单一进程中运行，并且 Internet 服务通常由其他进程组成。为此，Internet 服务与其他 Windows 服务不同，可以高效地关闭和重新启动。关于使用 IISReset.exe 的详细信息，请参阅重新启动 IIS。

  7. 为什么站点的一些访问者不能获得专门为他们的浏览器设计的 Web 页？

    8. 如果使用浏览器性能组件来决定客户查看能力，当更新版本的浏览器发布之后，就应该周期性地更新服务器上的文件 Browscap.ini。既可以从 Windows 2000 CD 盘中获得更新后的 Browscap.ini，也可以从以下站点 http://www.microsoft.com 下载。

  8. 在哪里可找到为残障用户提供的辅助特性的信息？

    9. 关于 Windows 辅助功能的常规信息，请参阅 Windows 文档。关于与 IIS 相关的辅助功能信息，请参阅 IIS 的辅助功能。

  9. 如何获得有关 IIS 的技术支持？

    10. 对于技术支持，如果是以初始设备制造商 (OEM) 软件包的部件形式预安装了本软件，或者本软件已被预先包含在购买的个人计算机上，则必须与 PC 或服务器制造商联系以获得支持。请查阅 PC 或服务器随带文档，获取制造商的电话号码。

    如果以零售形式获得本产品，并且遇到的技术问题联机文档无法解答，请访问 Web 上的 Microsoft 产品支持服务。

  10. 是否可以更改 Web 站点的名称并将对旧站点的请求重新定向到新站点？

    11. 可以正确配置 Web 站点，使其对旧名称和新名称均作出相应，假定网络能够正确地将两种请求指向您的计算机。这样，访问者使用旧名称也可以访问您的站点，而且还会告之新的名称。有关如何分配名称的说明，请参阅命名 Web 站点。

  11. 如何支持在 HTTP 请求头中不传递主机名的老式浏览器？

    12. 可以向默认主页添加脚本，将旧的浏览器重新定向到正确的站点或者建议升级浏览器的网页（如果浏览器不接收 cookies）。关于如何检测浏览器类型以及如何相应进行重定向的指令，请参阅支持旧浏览器中的主机头名。

  12. 是否可以使用 MMC 中的 Internet 信息服务管理单元通过代理服务器远程管理站点？

    13. MMC 中的 IIS 管理单元专门用于本地管理正在运行 IIS 的计算机，或者管理 Intranet中的另一个 IIS 站点。要远程使用 IIS 管理单元，可以使用终端服务。

    要想通过 Internet 或代理服务器实现远程管理任务，可以使用基于浏览器的 Internet 服务管理器 (HTML)。详细信息，请参阅远程管理。

  13. 试图使用基于浏览器的 Internet Service Manager (HTML) 来远程管理服务器时，为什么某些 Web 站点并不出现？在运行 IIS 的计算机上运行 Internet 服务管理器 (HTML) 时，列出了所有 Web 站点。

    14. 您可以作为“操作者”而非“管理员”连接。另外，请确保您已经将正确端口号附加到管理站点。详细信息，请参阅远程管理。

  14. 如何才能知道驱动器是否设置为使用 NTFS？

    15. 在“Windows 资源管理器”中，选择驱动器并其打开属性页。在“常规”选项卡上，列出了文件系统类型。

  15. 如何将格式化为文件分配表 (FAT) 文件系统的驱动器或分区转换为 NTFS？

    16. 在命令行下，键入 convert drive letter:/FS:NTFS.NTFS 驱动器不能转换回 FAT 类型。详细信息，请参阅 Windows 文档。

  16. 是否可以设置 FTP 站点使不同用户拥有不同级别的访问权限？

    17. 为了使访问仅限于某些计算机，请选择 IIS 管理单元中的一个站点，打开其属性页，选择“目录安全性”选项卡，然后再单击“IP 地址及域名限制”段内的“编辑”按钮。可以允许和拒绝访问除指定的计算机或子网以外的所有计算机和子网。

    在 NTFS 文件系统中，也可以使用单个虚拟目录并设置 NTFS 权限，以便允许或拒绝不同 Windows 用户帐户进行各种不同的访问。

  17. 如何确认服务器证书是否已被附加到特定的 Web 站点？

    18. 选择站点并打开其属性页。在“目录安全性”属性页中，“安全通讯”下有一个“编辑”按钮。如果此按钮已启用，表明已有证书与 Web 站点关联。如果此按钮变暗，则表明需要将服务器证书与站点关联才能使用安全套接字协议层 (SSL) 功能。

  18. 是否可将多个服务器证书附加到一个 Web 站点？

    19. 否。每个 Web 站点只能与一个服务器证书关联。

  19. 是否可将同一服务器证书附加到多个站点？

    20. 是。但一个服务器证书则可与任意 Web 站点关联。

  20. 是否可将服务器证书附加到一个 FTP 站点？

    21. 否。FTP 站点不支持 SSL 功能。

  21. 是否应该创建服务器证书的副本？

    22. 是。服务器证书非常有用，它是服务器 SSL 功能的核心。要创建服务器证书的备份，请将整个证书复制到一张软盘上，然后再放到安全的地方。

  22. 如何知道有多少用户与我的 FTP 服务相连？

    23. 在 IIS 管理单元中，选择一个 FTP站点，打开其属性页，然后再单击“当前会话”按钮。详细信息，请参阅性能调整。
