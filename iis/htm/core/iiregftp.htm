FTP 服务注册表项目

除了服务特有的注册表项目，FTP 服务还将保留下列参数。

注册表路径：

HKEY_LOCAL_MACHINE\SYSTEM
\CurrentControlSet
 \Services
  \MSFTPSVC
   \Parameters

AnnotateDirectories
REG_DWORD
范围： 0、1
默认：0（禁用）
FTP 服务支持使用自定义消息对目录注释。注释文本保存在被注释的目录中的一个名为 ~ftpsvc~.ckm 的专用文件中。如果该文件存在于更改目录 (CWD) FTP 操作中的目标目录中，那么服务响应此操作的文件的内容。这为管理员给考虑的目录添加自定义消息提供了一个方法。默认情况下，服务配置为不发送注释文本。如果选择添加自定义消息，也应该创建注释文件。同时，建议将注释文件设置为隐藏文件，这样文件就不会出现在目录列表中。

 

EnablePortAttack
REG_DWORD
范围： 0、1
默认：0（禁用）
该参数为默认设置，用来防止 FTP 协议规范中的安全问题。FTP 服务规范允许根据客户给的端口地址建立被动的连接。这将允许怀有恶意的用户执行 FTP 服务中的破坏性命令。当 FTP 服务使用一个不是 FTP 数据端口 (20) 的端口并且端口号小于 IP_PORT_RESERVED (1024) 的连接时，将出现这个问题。EnablePortAttack 控制是否应该允许这样的攻击。默认情况下，服务不建立任何到端口号小于 IP_PORT_RESERVED （20 除外）的连接。如果想要用户通过使用 FTP RFC 中指定的其他端口连接，应该启用该标记。

 

LowercaseFiles
REG_DWORD
范围： 0、1
默认：0（禁用）
FTP 服务使用文件名的原来格式（即按照保存在文件系统中文件名的格式，保留原来的大小写）。但是，为了与区分大小写的文件系统进行精确比较，可能必须确保使用了正确的文件名。管理员可以添加该值以确保服务为这些比较使用小写字母。
