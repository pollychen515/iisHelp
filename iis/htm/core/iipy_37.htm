目录安全属性表

使用此属性表可以设置 Web 服务器的安全特性。

匿名访问和身份验证控制

要配置 Web 服务器的身份验证和匿名访问特性，可单击 “编辑”。使用这些特性可以配置 Web 服务器，使其在授予用户对受限内容的访问权限之前先验证用户身份。

在服务器验证用户身份之前，必须首先创建有效的 Windows 用户帐户，并为这些帐户配置 Windows 文件系统 (NTFS) 目录和文件权限。

IP 地址和域名限制

（此特性只适用于 Windows 2000 Server。）

要允许或防止特定用户、计算机、计算机组或域访问此 Web 站点、目录或文件，可单击“编辑”。

安全通信

要配置 Web 服务器的安全套接字层 (SSL) 安全通信特性，可单击“编辑”。只有在被管理的计算机上安装了有效的服务器证书，才能使用 Web 服务器安全通信特性。

使用安全通信特性可以：

  • 要求用户通过建立安全（加密）链接来连接到目录或文件。
  • 设置 Web 服务器的客户证书映射和身份验证特性。
  • 启用或禁用客户证书映射
