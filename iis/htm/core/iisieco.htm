#echo

#echo 命令指示 Web 服务器将环境变量值插入到 HTML 页中。必须使用 HTML 注释定界符将指令括起来。

该指令仅可用于 HTML 页而无法用于 ASP 页。对于 ASP 脚本，可以通过使用Response.ServerVariables 方法访问环境变量（请参阅 ASP 内置对象 ）。

语法

<!-- #echo var = VariableName -->

参数

VariableName

指定希望插入其值的变量名。变量可以是下列各项之一：

变量 含义
ALL_HTTP 尚未解析成该表中描述的其他变量的所有 HTTP 头。这些变量的格式为 HTTP_头字段名。 HTTP 头由带有换行符分隔的单独 HTTP 头的空结束字符串组成。
AUTH_TYPE 这包含所使用的验证的类型。例如，如果使用基本验证，则字符串为“Basic”，而对于集成验证则为“integrated Windows authentication”。其他验证方案拥有其他字符串。因为新验证类型可以添加到 Web 服务器，所以不可能列出所有可能的字符串。如果字符串为空，则不使用验证。
AUTH_PASSWORD 在客户端身份验证对话框中输入的值。只有使用基本验证时，该变量才有效。
AUTH_USER 在客户端身份验证对话框中输入的值。
CONTENT_LENGTH 脚本可以期望从客户端接收到的字节数量。
CONTENT_TYPE POST 请求正文中提供的信息内容类型。
DOCUMENT_NAME 当前文件名。
DOCUMENT_URI 当前文档的虚拟路径。
DATE_GMT 以格林威治标准时间 (GMT) 表示的当前日期。
DATE_LOCAL 以本地时区表示的当前日期。
GATEWAY_INTERFACE Web 服务器使用的 CGI 规范修订本。以 CGI/revision 格式返回修订本。
HTTP_ACCEPT 特殊情况下的 HTTP 头。以逗号 (,) 连接和分隔接受字段的值。例如，如果以下行是 HTTP 头的一部分：

accept:*/*; q=0.1
accept:text/html
accept:image/jpeg

HTTP_ACCEPT 变量将具有值：

*/*; q=0.1, text/html, image/jpeg
LAST_MODIFIED 上一次修改当前文档的日期。
PATH_INFO 附加的路径信息，与客户端给定的一样。如果有的话，这是由脚本名称之后、查询字符串之前的 URL 后缀部分组成。
PATH_TRANSLATED 这是 PATH_INFO 的值，但具有扩展至目录规范的虚拟路径。
QUERY_STRING 跟随在引用该脚本的 URL 中问号 (?) 之后的信息。
QUERY_STRING_UNESCAPED 查询字符串的 Unescaped 版本；即不是 URL 编码的另一版本。
REMOTE_ADDR 发送请求的客户或客户代理（例如，网关或防火墙）的 IP 地址。
REMOTE_HOST 发送请求的客户或客户代理（例如，网关或防火墙）的主机名。IIS 2.0 和 3.0 为该参数返回一个 IP 地址。
REMOTE_USER 它包含由客户提供和服务器验证的用户名。当用户匿名（但经过验证）时，它将作为空字符串返回。
REQUEST_METHOD HTTP 请求方法。
SCRIPT_NAME 正在执行的脚本程序名。
SERVER_NAME 服务器的主机名或 IP 地址，就像其出现在自引用 URL 中那样。
SERVER_PORT 接收请求的 TCP/IP 端口。
SERVER_PORT_SECURE 是 0 或 1 的一个字符串。如果正在安全端口上处理请求，则其为1。否则将为 0。
SERVER_PROTOCOL 与该请求相关的信息检索协议的名称和版本。通常为 HTTP/1.0。以“名称/版本”格式返回该协议。
SERVER_SOFTWARE 应答请求的 Web 服务器的名称和版本。以“名称/版本”格式返回服务器信息。
URL 给定 URL 的基本部分。不包括参数值。当 Web 服务器从头中解析 URL 时，将确定该值。
