获取服务器证书

有两种方式可以获取服务器证书。可以颁发自己的证书，或者从证书颁发机构获得证书。下面的步骤解释如何在服务器上实施证书。关于证书的详细信息，请参阅关于证书。

要点 保护好证书和密钥对非常重要；必须将它们备份到磁盘并保存在安全的地方。

  • 发布自己的服务器证书
  • 从证书颁发机构获取服务器证书
  • 创建服务器证书和私钥的备份副本
发布自己的服务器证书

当决定是否颁发自己的服务器证书时需考虑以下问题：

  • 考虑证书服务提供的功能；Microsoft Certificate Services 2.0 包含不同的证书格式，并提供与证书相关的活动的审核或日志。
  • 比较颁发自己的证书和从证书颁发机构购买证书两者之间的费用。
  • 在了解、实施和将现有的安全系统和策略与证书服务集成之前，您的公司可能需要一个原始调整阶段。
  1. 使用证书服务创建用来发布和管理证书的自定义服务。可以为 Internet 或公司 Intranets 创建服务器证书，以便公司能够完全控制证书管理策略。详细信息，请参阅 Microsoft 证书服务文档。
  2. 使用 Web 服务器证书向导可以请求和安装服务器证书。

注意

  • 服务器证书的联机请求只能用于企业证书服务。IIS Web 服务器证书向导不能识别在同一台计算机中的单独的证书服务。使用脱机证书请求将请求保存到文件之中，然后再将其作为脱机请求进行处理（请参阅证书服务文档）。使用本地企业证书服务的联机注册不受影响。
  • 如果打开 SGC 证书，您可能会在“常规”选项卡上收到这样的通知：“证书无法检验其所有的预定用途”。该通知的签发是由于 SGC 证书与 Windows 2000 交互的方式而导致的。但该通知并不表示证书无法正常工作。

 

从证书颁发机构获取服务器证书

注意 如果正在替换当前的服务器证书，IIS 将继续使用旧证书直到新的请求完成。

  1. 找到能够为您的商业需要提供满意服务的证书颁发机构，然后请求服务器证书。

    2. 选择证书颁发机构时请考虑以下几方面的问题：

    • 证书颁发机构是否能够向您颁发证书，并使证书与访问服务器的所有浏览器兼容？
    • 证书颁发机构是否为经过验证的可信任的机构？
    • 证书颁发机构如何验证您的标识？
    • 证书颁发机构是否具有接收联机证书请求（如用 Web 服务器证书向导生成的请求）的系统？
    • 证书的初始费用以及更新或其他服务的费用是多少？
    • 证书颁发机构是否熟悉您所在组织或公司的利益？

    要了解支持 Internet 信息服务的证书颁发机构最新列表，请访问位于http://backoffice.microsoft.com/securitypartners/ 的 Microsoft Security Web 站点。在“按照分类”列表中，选择“证书颁发机构服务”。

  2. 使用 Web 服务器证书向导来创建证书请求，该请求将发送到证书颁发机构。
  3. 将请求发送到证书颁发机构。它们将处理请求并发送证书给您。

    4. 注意 在处理请求或颁发证书之前，某些证书颁发机构要求您证明自己的身份。

  4. 使用 Web 服务器证书向导来安装证书。

创建服务器证书和私钥的备份副本

注意 在以前的 IIS 版本中，使用私钥管理器来备份服务器证书。在此 IIS 版本中，Web 服务器证书向导代替了私钥管理器。因为 IIS 工作时与 Windows 关系密切，所以可以使用证书管理工具导出或备份服务器证书。

要备份服务器证书
  1. 定位正确的证书存储位置。这通常是证书管理器中的本地计算机存储位置。

    2. 注意 如果没有在 MMC 中安装证书管理器，则需要安装。

  2. 选择个人存储位置中的证书。
  3. 打开“动作”菜单，指向“所有任务”，并单击“导出”。
  4. 在“证书管理器导出向导”中，选择“是，导出私钥”。
  5. 保留向导默认设置，当出现提示时输入证书备份文件的密码。

    6. 要点 不要选择“如果导出成功，删除密钥”，因为这将禁用当前服务器证书。

  6. 完成向导来导出服务器证书的备份副本。
要将证书管理器添加到 MMC

注意 如果已经在 MMC 中安装了证书管理器，它将指向正确的本地计算机证书存储位置。

  1. 打开 MMC 控制台并从“控制台”菜单中选择“添加/删除管理单元”。
  2. 单击“添加”。
  3. 选择“证书管理器”。
  4. 单击“添加”。
  5. 选择“计算机帐户”选项。
  6. 选择“本地计算机”选项。
  7. 单击“完成”。
