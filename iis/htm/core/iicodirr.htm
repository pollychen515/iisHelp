将请求重定向到目录

可以将目录上的文件请求重定向到其他目录、Web 站点或其他不同目录上的文件。当浏览器请求原始 URL 上的文件时，Web 服务器指导浏览器使用新的 URL 请求页。

重定向请求到其他目录或 Web 站点
  1. 在 Internet 信息服务管理单元中，选中 Web 站点或目录，然后打开其属性表。
  2. 单击“主目录”、“虚拟目录”或“目录”选项卡。
  3. 选中“重定向到 URL”。
  4. 在“重定向到”框中，键入目标目录或 Web 站点的 URL。例如，要将 /Catalog 目录中的文件请求重定向到 /NewCatalog 目录中，请键入“/NewCatalog”。

 

将所有请求重定向到单个文件中
  1. 在 Internet 信息服务管理单元中，选中 Web 站点或目录，然后打开其属性表。
  2. 单击“主目录”、“虚拟目录”或“目录”选项卡。
  3. 选中“重定向到 URL”。
  4. 在“重定向到”框中，键入目标文件的 URL。
  5. 选中“上面输入的准确 URL”以防止 Web 服务器将原始文件名添加到目标 URL 中。

可以在目标 URL 中使用通配符和重定向变量精确地控制如何将原始 URL 转化成目标 URL。详细信息，请参阅重定向参考。
