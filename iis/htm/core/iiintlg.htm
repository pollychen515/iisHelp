日志记录属性参考

本主题列出了日志记录属性的定义和用于 W3C 扩充日志记录格式的值。分为下列部分：

  • W3C 扩充日志记录定义：W3C 扩充日志文件格式可用的常规属性。
  • 处理帐户日志记录定义：处理 W3C 扩充日志文件中出现的帐户的其他属性。

有关 W3C 扩充日志格式的详细信息，请参阅位于 http://www.w3.org/ 的 W3C 的 Web 站点。

W3C 扩充日志记录定义

前缀 含义
s- 服务器操作
c- 客户机操作
cs- 客户机到服务器的操作
sc- 服务器到客户机的操作


字段 出现 说明
日期 date 执行操作的日期。
时间 time 活动发生的时间
客户机 IP 地址 c-ip 访问服务器的客户机 IP 地址。
用户名 c-username 访问服务器的已验证用户的名称。这不包括用连字符表示的匿名用户。
服务名称和实例编号 s-sitename Internet 服务和在客户计算机上运行的实例编号。
服务器名称 s-computername 生成日志条目的服务器名称。
服务器 IP s-ip 生成日志条目的服务器 IP 地址。
方法 cs-method 客户机试图执行的操作（例如 GET 方法）
URI 词干 cs-uri-stem 访问的资源；例如 Default.htm。
URI 查询 cs-uri-query 查询，如果有的话客户机将试图执行。
Http 状态 sc-status 以 HTTP 术语表示的操作的状态。
Win32 状态 sc-win32-status 以 Windows 2000 使用的术语表示的操作的状态。
发送字节数 sc-bytes 服务器发送的字节数。
接收字节数 cs-bytes 服务器接收的字节数。
服务器端口 s-port 客户机连接的端口号。
所用时间 time-taken 操作所占用的时间。
协议版本 cs-protocol 客户机使用的协议 (HTTP、FTP) 版本。HTTP 协议应为 HTTP 2.0 或 HTTP 2.1。
用户代理 cs(User-Agent) 客户机使用的浏览器。
Cookie cs(Cookie) 发送或接收的 cookie 内容（如果有的话）。
引用者 cs(Referer) 用户访问的前一个站点。此站点提供与当前站点的链接。

处理帐户日志记录定义

字段 出现 说明
进程类型 s-proc-type7 触发事件的进程类型，CGI 或进程外应用程序。类型可以是 CGI、应用程序或全部。
进程事件 s-event 触发的事件：Site-Stop、Site-Start、Site-Pause、Periodic-Log、Interval-Start、Interval-End、Interval-Change、Log-Change-Int/Start/Stop、Eventlog-Limit、Priority-Limit、Process-Stop-Limit、Site-Pause-Limit、Eventlog-Limit-Reset、Priority-Limit-Reset、Process-Stop-Limit-Reset 或 Site-Pause-Limit-Reset。有关这些值的说明，请单击这里。
用户时间总计 s-user-time 站点在当前间隔使用的累计用户模式处理器时间总和（以秒为单位）。
内核时间总计 s-kernel-time 站点在当前间隔使用的累计内核模式处理器时间总和（以秒为单位）。
页错误总计 s-page-faults 导致内存页错误的内存引用总数。
进程总计 s-total-procs 在当前间隔内创建的 CGI 和进程外应用程序总数。
活动进程 s-active-procs 记录日志时运行的 CGI 和进程外应用程序的总数。
终止进程总数 s-stopped-procs 由于在当前间隔内的进程限制而停止的 CGI 和进程外应用程序的总数。

值 含义
Site-Stop 由于某种原因而停止的 Web 站点。
Site-Start 启动或重新启动的 Web 站点。
Site-Pause 暂停的 Web 站点。
Periodic-Log 这是一个有规律定义的日志条目，其间隔由管理员指定。
Interval-Start 重新设置的间隔已经开始。
Interval-End 已达到重新设置的间隔并重置。
Interval-Change Web 站点管理员更改了重新设置间隔的值。
Log-Change-Int/Start/Stop 发生了下列事件之一：日志的间隔已经更改；发生了间隔事件或者站点停止、启动或暂停。
Eventlog-Limit 由于 CGI 或进程外应用程序达到管理员设置的事件日志限制，生成一个 Web 站点的事件日志。
Priority-Limit 由于 Web 站点达到了管理员设置的低优先级限制，该站点将 CGI 或进程外应用程序设置为低优先级。
Process-Stop-Limit 由于达到了管理员设置的进程停止限制，Web 站点停止 CGI 或进程外应用程序。
Site-Pause-Limit 由于 CGI 或进程外应用程序达到了管理员设置的站点暂停限制，Web 站点暂停。
Eventlog-Limit-Reset 已经达到“重新设置间隔”，或者手工重新设置了 Eventlog-Limit。
Priority-Limit-Reset 已经达到“重新设置间隔”，或者手工重新设置了 Priority-Limit。
Process-Stop-Limit-Reset 已经达到“重新设置间隔”，或者手工重新设置了 Process-Stop-Limit。
Site-Pause-Limit-Reset 已经达到“重新设置间隔”，或者手工重新设置了 Site-Pause-Limit。
