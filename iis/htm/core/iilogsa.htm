记录站点活动

启用 Web 站点日志可以收集有关用户活动的信息。信息存储在 ASCII 文件或 ODBC 相容数据库中。IIS 的记录信息日志功能不在 Windows 2000 的事件日志或性能监视功能之列。日志中的信息包括站点的访问者、访问者查看的内容以及最后一次查看信息的时间等。可以使用日志来评估内容受欢迎程度或识别信息瓶颈。

本节包括：

  • 关于记录站点活动：关于日志、日志格式的描述、日志文件大小的讨论和创建新的日志文件的概述。
  • 启用日志记录：启用或禁用 Web 或 FTP 站点或目录的日志。
  • 跟踪处理器使用：设置进程帐户以确定进程外应用程序对 CPU 的使用情况。
  • 自定义 W3C 扩充日志记录：指定 W3C 扩充日志文件中所包括的字段。
  • 保存日志文件：指定日志文件目录和用于创建新日志文件的选项。
  • 将日志文件转换为 NCSA 格式：使用命令行实用程序 Convlog.exe 将日志转换为 NCSA 通用日志文件格式。

本节不包括：

  • 要使用监视工具而不使用日志，请参阅性能调整。
  • 要生成、查看或了解安全日志（审核），请参阅安全。
