Web 站点管理

无论站点位于 Intranet 还是 Internet，提供内容的原理是一样的。首先您必须将您的 Web 文件放入服务器的目录中，这样用户就可以建立 HTTP 连接并使用 Web 浏览器查看您的文件。但是，除了仅仅在您的服务器上存储文件之外，还必须管理您的站点的配置方法，更重要的是如何发展您的站点。目前，吸引人的 Web 站点很少是一些页面的静态集合。最成功的 Web 管理员总是忙于适应不断变化的 Web 内容。

在这一部分中您将学习管理 Web 站点基本结构的基础知识，从重新定向请求到动态改变 Web 页。

本节包括：

  • 关于 Web 站点管理:简单介绍组织和管理 Web 站点的过程。
  • 选择创作工具:如何选择满足需要的创作工具的概述
    • .
  • 使用 ASP 管理 Web 站点内容:对 Microsoft Active Server Pages (ASP) 和其帮助管理 Web 内容的方法的介绍。
  • 更改主目录:指定站点的主要位置。
  • 创建虚拟目录:将目录添加到站点。
  • 将请求重定向到目录:将浏览器请求发送到一个新位置。
  • 重定向程序请求:将浏览器请求发送给应用程序。
  • 启用内容截止日期:配置时间敏感的信息设置。
  • 使用内容分级:对站点内容进行分级以帮助用户避免令人反感的材料。
  • 为 Web 页添加页脚:将页脚自动插入到站点的每个 HTML 页。
  • 服务器端包含:将服务器命令插入到 Web 页，Web 页是在请求该页之后但在发送到浏览器之前处理的。

本节不包含：

  • 使用 ASP 创建服务器端脚本的基础；请参阅 Active Server Pages。
  • 添加 Web 站点；请参阅 Web 和 FTP 站点。
