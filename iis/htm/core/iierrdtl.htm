启用详细自定义错误消息

IIS 拥有一套自己独特的自定义错误，提供的反馈量较默认的 HTTP 1.1错误消息更大，而且更为“友好”。例如，默认的 HTTP 1.1 404 错误消息所包含的“找不到对象”语句已被扩展为：“Web 服务器找不到要求的文件/脚本。请检查 URL 以确保路径正确。如果问题仍然存在，请与服务器管理员联系。”默认情况下，在 IIS 管理单元的“默认Web站点”中设置这些详细的自定义错误消息。

配置详细的错误消息。
  1. 在 Internet 信息服务管理单元中，选择要在其中自定义 HTTP 错误消息的 Web 站点、虚拟目录、目录或文件，然后单击“属性”按钮。
  2. 选择“自定义错误”属性表。
  3. 选择要更改的默认 HTTP 错误消息。
  4. 单击“编辑属性”按钮。
  5. 在“消息类型”框中，选择“文件”。
  6. 单击“浏览”并导航到 drive letter:\WINNT\help\iisHelp\common\文件名。

默认情况下，友好错误消息被安装到以下位置：drive letter:\WINNT\Help\iisHelp\common。文件名是与特定 HTTP 错误对应的数字；例如，400.htm、401-1.htm 等等。

自定义错误安装位置―IIS 4.0 到 IIS 5.0

如果从 IIS 4.0 升级到 IIS 5.0，并且以前创建了自己的自定义错误文件或者修改过 IIS 4.0 安装在 drive letter:\WINNT\Help\common directory 中的自定义错误文件，则自定义错误文件就会被移动到 drive letter:\WINNT\Help\iisHelp\common directory 中，且文件扩展名被更改为 .bak，“drive letter”表示驱动器盘符。可以检索自定义错误并在 IIS 5.0 中重新使用。例如，IIS 4.0 安装的自定义错误文件 400.htm 已被更名为 400.bak，位于 drive letter:\WINNT\Help\iisHelp\common directory （除了新的 IIS 5.0 自定义错误文件外）。
