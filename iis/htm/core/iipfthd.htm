限制进程

如果在一台计算机上运行多个 Web 站点，或在作为 Web 服务器的计算机上运行其他应用程序，您可以限制允许 Web 站点外部进程应用程序使用的处理器时间。这样就可以确保其他 Web 站点或非 Web 应用程序的处理器使用时间。

注意 如果限制站点中的虚拟目录配置为进程内部或共用进程应用程序，则站点可能实际上超过了处理器使用限制。进程内部和共用进程应用程序不受处理器限制的影响，而且也不包含在进程帐号统计信息中。

要决定是否应该使用处理器限制
  1. 使用系统监视器检查 Processor 对象的“% Processor Time”计数器、以及 Web 服务对象中用于“最大 CGI 请求”和“总计 CGI 请求”的特定实例计数器。
  2. 启用进程帐号，以便在 IIS 日志中包括“Job Object”计数器报告。
  3. 检查 DLLhost 对象计数器以确定外部进程 WAM 和 ISAPI 请求的数量。
要限制 Web 站点使用的处理器
  1. 在 Internet 信息服务管理单元中， 选择要限制使用处理器的 Web 站点。
  2. 打开 Web 站点的属性页，然后单击“性能”选项卡。
  3. 选择“启用进程限制”并键入限制该站点占用 CPU 时间的百分比。
  4. 如果要启用处理器时间超出限度的 Level 2 和 Level 3 结果，请选择“强制性限制”。
