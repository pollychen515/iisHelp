启用服务器端包含

在处理文件前，Web 服务器必须检测到文件包含有服务器端包含命令。默认情况下，设置 Web 服务器用来处理带有.stm、.shtm 或 .shtml 文件扩展名的文件。可以为整个 Web 服务器、Web 站点、应用程序或单独目录禁用服务器端包含命令的处理。

启用服务器端包含
  1. 为包含 SSI 命令的文件指定 SSI 文件扩展名（.stm、.shtm 或 .shtml)。

    2. 可以添加附加的扩展名；请参阅设置应用程序映射。

  2. 将 SSI 文件放入具有“脚本”或“执行”访问权限的目录中。详细信息，请参阅设置目录或文件的 NTFS 权限。

禁用服务器端的包含
  1. 在 Internet 信息服务管理单元中， 选择一个 Web 站点或目录，打开其属性页，然后单击“主目录”或“目录”选项卡。要禁用应用程序的 SSI，选择应用程序起始点目录。
  2. 单击“配置”按钮，然后单击“应用程序映射”选项卡。
  3. 选择扩展名，然后单击“删除”按钮。当 Web 服务器接收到对带有该文件扩展名的页的请求时，将不在该页中处理服务器端包含命令。
