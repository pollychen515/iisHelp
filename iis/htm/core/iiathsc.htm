验证

可以配置 IIS 进行“验证”，或在允许该用户与服务器建立网络连接前确定用户的 Windows 用户帐户标识。但是，用户验证仅在禁用匿名访问，或在 NTFS 权限要求用户通过有效 Windows 用户帐户、用户名和密码证明身份时进行。

凭借 IIS 提供的验证选项，您可以选择同时满足安全需要和所用 Web 浏览器能力的验证方法。

本节包括：

  • 关于验证： 对验证的概述。
  • 启用和配置验证： 启用和配置基本、摘要式和集成 Windows 三种验证方法的过程。
  • 设置默认登录域： 配置基本验证登录域的过程。
  • 在 IIS 中使用 Fortezza 安全机制：在 IIS 中配置 Fortezza 安全功能的过程。

本节不包含：

  • 控制用户访问您的 Web 服务器文件和目录的方法；请参阅访问控制。
  • 使用 SSL 或证书，请参阅证书。
