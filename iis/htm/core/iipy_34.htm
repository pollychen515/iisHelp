应用程序映射属性表

使用此属性表可以映射文件扩展名到用来处理这些文件的程序或解释器。映射的应用程序包括 Internet Server API (ISAPI) 应用程序、Active Server Pages (ASP) 应用程序、Internet Database Connector (IDC) 应用程序以及使用服务器端包含命令的文件。例如，当 Web 服务器收到对带有 .asp 文件扩展名的页的请求时，将使用应用程序映射来决定调用可执行文件 asp.dll 处理该页。

高速缓存 ISAPI 应用程序

可以加载并高速缓存 Internet Server API DLLs，以便处理进一步的请求时不再调用应用程序。大多数 ISAPI 应用程序（包括 Active Server Pages）都受益于高速缓存。只有在特定环境下，如调试 ISAPI 应用程序，才可以清除该选项。

如果服务器上有多个 Web 站点加载并高速缓存了同一个 ISAPI 应用程序，则对服务器清除此选项不会从内存中卸载此应用程序。必须对使用此应用程序的所有 Web 站点清除此选项。清除此选项不会卸载正在执行的应用程序。只有后续请求不被高速缓存。

应用程序映射

此表列出了与可执行文件相关联的文件扩展名和可执行文件的名称。如果“（所有）”出现在列表中，则所有动作发送给应用程序；“（所有）”不是动作。

要点 在 IIS 4.0 的“应用程序映射“属性表中，在“排除”内有动作列表。IIS 5.0 对此作了更改，在“动词”列中列出了应用程序运行的 HTTP 动作。

要添加应用程序映射

  1. 单击“添加”。
  2. 在“Ext”列中输入可执行文件的扩展名。
  3. 在“可执行路径”列中输入可执行文件的物理路径。
  4. 输入可执行文件应传递给应用程序的 HTTP 动作。要指定将所有动作传递给应用程序，则保留此字段为空（单击“确定”后此字段将装入“（所有）”）。方法名之间用英文逗号 (,) 分隔。
  5. 如果要让应用程序在没有“执行”权限的目录内运行，可选择“脚本引擎”复选框。
  6. 选择“检查文件”复选框，可以指示 Web 服务器验证请求的脚本文件是否存在，并保证请求用户有访问此脚本文件的权限。
  7. 单击“确定”。

要删除应用程序映射，选择扩展名并单击“删除”。

注释

  • “脚本引擎”设置主要是用于基于脚本的应用程序，如映射到解释器的 ASP 和 IDC。要运行脚本映射的应用程序，必须为应用程序所在的目录选中“脚本”或者“执行”访问复选框。要只允许运行脚本映射的应用程序，可使用“脚本”访问。要允许运行脚本映射的应用程序和可执行文件（.exe 和 .dll），可使用“执行”访问。
  • 如果选中了“检查文件”复选框，但脚本不存在或者用户没有访问此脚本文件的权限，则相应的警告消息返回浏览器并且脚本引擎不会激活。此选项对于映射到非 CGI 可执行文件的脚本非常有用，这些可执行文件（如 Perl 解释器）无法在脚本不能访问时发送 CGI 响应。由于脚本会打开两次，服务器和脚本引擎各打开一次，所以启用此选项会对性能有一定影响。
