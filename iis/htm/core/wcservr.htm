创建发布目录

下面逐步介绍建立发布目录，即 WebDAV 目录的步骤。

建立发布目录
  1. 在 Windows 2000 桌面上，单击“我的电脑”。
  2. 在 Inetpub 目录下，创建一个实际的目录。

    3. 例如，如果创建的目录称为 WebDAV，到该目录的路径就类似于：C:\Inetpub\WebDAV

    可以将该目录放到除 Wwwroot 目录以外的任何地方。Wwwroot 是一个例外，因为其默认 DACL 与其他目录上的不同。

  3. 在 IIS 管理单元中，创建一个虚拟目录。

    4. 有关信息，请参阅创建虚拟目录。

  4. 键入“WebDAV”，作为虚拟目录的别名，并将该目录与在第 2 步中创建的实际目录链接。
  5. 并授予该虚拟目录的“读”、“写”、“浏览”权限。

    6. 这样就授予了用户在该虚拟目录上发布文档以及查看该目录下文件列表的权限。虽然因安全原因并不推荐这样做，但您可以将同样的访问权限授予整个 Web 站点并允许客户发布到整个 Web 服务器中。

注意 授予写访问权限并不授予客户修改 Active Server Pages (ASP) 或其他任何脚本映射文件的能力。要允许这些文件可以被修改，必须在创建虚拟目录之后授予写权限和脚本源访问权限。有关设置这些权限的信息，请参阅设置 Web 服务器权限。

一旦您完成了 WebDAV 虚拟目录的设置，就可以允许客户发布到上面。有关用户如何通过任意 Microsoft WebDAV 客户端连接到该目录的信息，请参阅发布和管理文件。
