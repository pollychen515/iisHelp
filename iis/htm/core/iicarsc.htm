使用证书信任列表

可以配置服务器接受预定义证书授权机构 (CA) 列表中的证书。可以配置每个 Web 站点通过使用证书信任列表 (CTLs) 接受不同列表中的证书。可以根据 CTL 验证客户端证书。可以使用 CTL 向导创建和编辑 CTL，并将新的根证书添加到自己的 CTL 中。

例如， Intranet 管理员可以通过 CTL 的形式为 Intranet 上各个部门的 Web 站点创建不同的信任证书颁发机构列表。IIS 将只接收该部门的 CTL 上列出的证书颁发机构的证书。当该部门的成员使用部门 CTL 中某个证书颁发机构中的证书登录时，他们将被自动验证。

注释

  • 最常用的证书颁发机构根证书已经由 IIS 安装了。
  • CTL 只在 Web 站点级应用，不能应用在虚拟目录或文件上。
  • CTL 不可用于 FTP 站点。
在服务器上创建证书信任列表。
  1. 请使用管理员帐户登录到您的 Web 服务器。

    2. 要点 您必须按照下面的步骤使用管理员帐户登录。无法使用其他属于 Windows 管理员组的帐户。

  2. 在 Internet 信息服务管理单元中，打开 Web 站点的属性表。
  3. 在“目录安全性”属性表的“安全通信”下，单击“编辑”。

    4. 注意 只有将服务器证书附加或绑定到站点，“编辑”按钮才可激活。有关获取和安装服务器证书的信息，请参阅使用新的安全任务向导。

  4. 在“安全通信”对话框中，选中“启用证书信任列表”复选框，然后单击“新建”。CTL 向导将在创建 CTL 的整个过程中指导您操作。
修改服务器上的 CTL。
  1. 使用管理员帐户登录到 Web 服务器。
  2. 在 Internet 信息服务管理单元中，打开 Web 站点的属性表。
  3. 在“目录安全性”属性表的“安全通信”下，单击“编辑”。
  4. 在“安全通信”对话框中，选中想要修改的的 CTL，然后单击“编辑”。CTL 向导将开始启动并在修改 CTL 的整个过程中指导您操作。

注意 如果使用 Microsoft Certificate Services 2.0 来颁发证书，必须同时安装根证书。详细信息，请参阅 Microsoft 证书服务文档。
