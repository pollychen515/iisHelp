限制带宽

如果 Web 服务器使用的网络或 Internet 连接也被其他服务使用，例如电子邮件 或新闻，则可能希望限制 Web 服务器所使用的带宽以便其他服务也可使用。如果运行多个 Web 站点，则可以单独限制每个站点使用的带宽。只能限制静态 HTML 文件使用的带宽。如果使用套接字池，则限制一个站点的带宽会限制共享该端口号的所有其他站点。默认情况下，启用套接字池。关于禁用套接字池的详细信息，请参阅“关于内存管理”中的平衡内存使用和响应速度。

确定服务器使用的带宽
  1. 使用系统监视器检查 Network Interface 对象中的 Bytes Total/sec 或 Current Bandwidth 计数器。（如果希望比较传入和传出通讯量，可以检查 Bytes Sent/sec 和 Bytes Received/sec。）
  2. 将这些值与网络连接的全部带宽进行比较。在负载正常的情况下，服务器使用的带宽不应超过其全部可用带宽的 50%。剩下的“净空”带宽在高峰期间使用。

    注意 如果服务器容易遭受极大高峰或极度阻碍，则将正常负载保持在总量的 50% 之下不失为一个好办法。

限制 IIS 使用的带宽
  1. 在 Internet 信息服务管理单元中， 选择运行 IIS 的计算机。
  2. 在“Internet 信息服务”属性页中，选择“启用带宽限制”复选框。
  3. 在“最大网络使用”框中，键入希望 IIS 使用的每秒千字节 (KBps) 的最大值。

 

限制单独 Web 站点使用的带宽
  1. 在 Internet 信息服务管理单元中，选择 Web 站点，并单击“属性”按钮显示其属性页。
  2. 在“性能”属性页中，选择“启用带宽限制”。
  3. 在“最大网络使用”框中，键入希望站点使用的每秒千字节的最大值。
