身份验证方法

使用该对话框配置 Web 服务器来检验用户的标识，也叫“验证”，以防止未经授权的用户建立到受限制的内容的 Web (HTTP) 连接。验证过程包括确定用户是否拥有访问特定 Web 站点、目录或文件的适当的 Windows 文件系统 (NTFS) 权限的有效 Windows 用户帐户。

匿名访问

要允许用户建立匿名连接，请选择该选项。匿名或 Guests 帐户是有效的 Windows 用户帐户，服务器将登录使用这些帐户的用户。

验证访问

基本验证

要启用 Web 服务器的基本验证方法（它是一种广泛用于识别用户的工业标准方法），请选择该选项。

要点 基本验证使得密码以未加密的形式通过网络传送。一个计算机肆意破坏者使用网络监视工具就可以截取用户名和密码。

编辑

要为那些没有明确提供给他们域名的用户将 Web 服务器配置为假设默认登录域，而不是本地域，请单击“编辑”。

摘要式验证：

该方法在 IIS 5.0 中是新的，它通过网络发送散列值而不是密码。这个方法越过了代理服务器和其他防火墙。

集成 Windows 验证

在集成 Windows 验证过程中，Web 服务器从事于使用用户的 Internet Explorer Web 浏览器进行密码信息交换。

一旦启用，Web 服务器在下列情况下将仅使用集成 Windows 验证：

  • 禁用匿名访问。
  • 拒绝匿名访问，因为 Windows 文件系统权限已设置，要求用户在建立与受限制的内容的连接之前提供 Windows 用户名和密码。
