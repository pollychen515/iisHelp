关于 Windows 事件

只要 Windows 启动，在“系统”、“安全性”和“应用程序”日志中自动开始日志记录。对于每种日志，可以筛选要查看的事件，指定要查看的项目数，决定储存项目的周期以及确定日志充满之后是否自动覆盖现有事件。

ASP 错误和 Windows 事件记录。

如果选中“进程选项”属性页上的“不成功的客户请求写入事件日志”复选框，ASP 错误就会被发送给 Windows 事件日志（在“应用程序日志”中）。此复选框对应于 metabase 属性 AspLogErrorRequests；也就是说，如果选中该复选框，AspLogErrorRequests 值就为 TRUE。如果不选中该复选框，AspLogErrorRequests 值就为 FALSE。

另一个 metabase 属性，即 AspErrorsToNTLog，决定向 Windows 事件日志发送哪个 ASP 错误AspErrorsToNTLog 的默认值为 FALSE。这意味着只有 ASP 错误的某个子集才被发送给 Windows 事件日志。如果 AspErrorsToNTLog 值为 TRUE，则所有 ASP 错误都被发送给 Windows 事件日志。

如果选中了“进程选项”属性页上的“不成功的客户请求写入事件日志”复选框，而且由于错误比较严重而导致 AspErrorsToNTLog 值为 FALSE，因此，就会将 ASP 错误发送给 Windows 事件日志：

100、101、102、103、104、105、106、107、115、190、191、192、193、194、240、241、242。

关于这些错误的描述，请参阅 Active Server Pages 事件。

不能通过 IIS 管理单元设置 AspErrorsToNTLog 值，而必须采取程序方式设置该属性。关于 metabase 以及采用程序方式管理 IIS 的详细信息，请参阅采取程序方式管理 IIS。

注意 如果将 ASP 错误记录到 IIS 日志文件时出现错误，这些错误将被发送给 Windows 事件日志。
