独立的应用程序

单个应用程序在与其他 Web 服务器和其他应用程序不同的进程或内存空间中运行。可以将应用程序隔离到三个应用程序保护级别中的一个。详细信息，请参阅“关于应用程序”主题中的应用程序保护。

有关性能问题的进一步讨论，请参阅关于性能调整。

注意 服务器端包含 (SSI) 和 Internet 数据库连接器 (IDC) 应用程序无法在 Web 服务器之外的内存空间运行。

设置或更改应用程序保护级别
  1. 在 Internet 信息服务管理单元中，选择 Web 站点或应用程序的开始点目录。
  2. 打开目录的属性表，然后单击“主目录”、“虚拟目录”或“目录”选项卡。

    3. 如果在列出为“开始点”目录的目录中，则需填写“应用程序名称”对话框。

  3. 在“应用程序保护”下拉菜单中，选择适当的处理选项。
  4. 单击“确定”。

    5. Web 服务器在创建单独的进程前，将完成任何当前的对应用程序请求的处理。在对应用程序的下一个请求中，它将在单独的内存空间运行。
