添加自定义错误消息

自定义错误消息可为两种形式：即映射为文件或者映射为 URL。这两种形式都可通过使用 IIS 管理单元中的“自定义错误”属性页实现。

采用映射为文件的形式自定义错误消息。

  1. 创建包含自定义错误消息的文件并将其放入某个目录中。
  2. 在 Internet 信息服务管理单元中，选择想在上面自定义 HTTP 错误的 Web 站点、虚拟目录、目录或文件，然后单击“属性”按钮。
  3. 选择“自定义错误”属性表。
  4. 选择想要更改的 HTTP 错误。
  5. 单击“编辑属性”按钮。
  6. 从“消息类型”框中选择“文件”。
  7. 键入指向自定义错误消息的路径和文件名，或使用“浏览...”。按钮定位计算机硬盘上的文件。
  8. 单击“确定”。

采用映射为 URL 的方式自定义错误消息。

  1. 创建包含自定义错误消息的文件并将其放入虚拟目录中。
  2. 在 Internet 信息服务管理单元中，选择想在上面自定义 HTTP 错误的 Web 站点、虚拟目录、目录或文件，然后单击“属性”按钮。
  3. 选择“自定义错误”属性表。
  4. 选择想要更改的 HTTP 错误。
  5. 单击“编辑属性”按钮。
  6. 从“消息类型”框中选择“文件”。
  7. 键入指向自定义错误信息的 URL，方式是输入以虚拟目录开头的 URL 路径。
  8. 单击“确定”。

注释

  • 如果定义了用于处理自定义错误的 ASP 文件，脚本将作为传送上下文的 URL 被调用。例如，如果有一个定义好的用于处理 HTTP 405 错误的 .asp 文件 Handle405.asp，一旦用户试图访问 Example.htm 时出现 405 错误，就会调用此.asp 文件，好象用户已经在 Web 浏览器中键入了 http://example.microsoft.com/handle405.asp?405;http://example.microsoft.com/example.htm。
  • 如果使用静态自定义错误文件（HTML 文件），应该总是使用“文件”选项。如果打算开发一个应用程序（使用 ISAPI 或 ASP）来处理错误，则应使用 URL 选项。应该注意，错误状态是用 URL 参数传递给应用程序的，而设置 HTTP 头状态则是应用程序的职责；否则，HTTP 的反应状态就是“HTTP 1.1 200 OK”。
  • 如果使用 Windows 随带的 Internet Explorer 5.0 ，则 Internet Explorer 将使用自己的 HTTP 错误代替以下自定义错误。如果自定义错误的文件小于列出的大小，就不会使用自定义错误：

    403、405 以及 410 - 必须大于 256 字节
    400、404、406、 408、409、500、500.12、500.13、500.15、501 以及 505 - 必须大于 512 字节
