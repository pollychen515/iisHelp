关于 Metabase

通过使用 IIS 管理单元和 Internet 服务管理器 (HTML) 等管理工具配置值，可以修改 metabase。还可以通过使用 IIS Admin 对象和 IIS Admin Base 对象配置值，从而以程序方式修改 metabase。

关于 metabase 结构、metabase 键和路径、属性继承以及 metabase 安全的详细信息，请参阅 IIS Metabase 简介。

关于使用 IIS Admin 对象和 IIS Admin Base 对象以程序方式管理 IIS 的信息，请参阅以程序方式管理 IIS。

关于 metabase 所有可配置属性的综合参考信息，请参阅管理属性参考。

  • Namespace
  • 继承

警告 在 metabase 中属性配置不当可能导致故障（包括 Web 站点或 FTP 站点失效）。如果出错，Web 站点或 FTP 站点的配置将损坏。仅可对无法在 IIS 管理单元或 Internet 服务管理器 (HTML) 中调整的设置编辑 metabase 属性，并应在直接编辑 metabase 时加以注意。

Namespace

命名空间指定 metabase 属性的位置。其组织如下：

LM/Service/Website/Root/virtual directory/dir/file

其中 LM = 本机
Service = Internet 服务 （W3SVC 或 MSFTPSVC)
Website = Web 站点
Root = 虚拟目录根
virtual directory = 虚拟目录
dir = 目录
file = 文件

例如，如果 metabase 路径的命名空间
LM/W3SVC/Website1/Root

与以下路径相关
C:\Inetpub\wwwroot

则 URL
http://domain.com/default.htm

将映射到物理文件路径
C:\Inetpub\wwwroot\default.htm。

继承

在较高级别（如 Web 站点级）上配置的 Metabase 属性可以传递或继承给较低的级别，如目录级。这些继承属性也可在较低级别上单独编辑。如果在单独的服务器、目录或文件上更改了属性，则在较高级别上更改的设置将不会自动覆盖较低级别的独立设置。
