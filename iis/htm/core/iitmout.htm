启用保持 HTTP 激活

大多数 Web 浏览器要求服务器在多个请求中保持连接打开。这就是保持 HTTP 激活。保持激活是一个 HTTP 规范，它能够显著增强服务器性能。如果没有它，浏览器将不得不为包含多个元素（如图形）的页产生大量的连接请求。将不得不为每个元素产生单独连接。这些额外的请求和连接要求额外的服务器活动和资源，这将会降低服务器的效率。还会使浏览器速度变慢并降低其响应程度。在安装进程中，将默认启用保持 HTTP 激活。

禁用保持 HTTP 激活
  1. 在 Internet 信息服务管理单元中，选择 Web 站点并打开其属性页。
  2. 在“Web 站点”属性页中，清除“启用保持 HTTP 激活”框。
