错误消息属性表

使用此属性表可以自定义 HTTP 错误，当 Web 服务器发生错误时，将发送 HTTP 错误给客户。管理员可以使用标准 HTTP 1.1 错误和 IIS 提供的详细自定义错误文件，或者创建自己的自定义错误文件。

要更改自定义错误信息的属性，选择 HTTP 错误，并从“类型”列中选择 “默认”、“文件”或者“URL”。

如果选择“默认”选项，将返回标准 HTTP 1.1错误给客户。默认 HTTP 1.1 错误的文本显示在“内容”列中。

如果选择“文件”选项，需输入自定义错误文件的本地路径。

如果选择“URL”选项，需输入以虚拟目录名开始的 URL 路径，如 /virdir1/errors/404.htm，此 URL 必须存在于本地服务器。
