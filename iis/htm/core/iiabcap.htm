网络容量

本节说明如何监视和调整与网络连接相关的一些属性值。一旦您对系统进行了调整，建议您继续使用活动日志和监视工具以提供性能反馈。

本节包括：

  • 关于容量计划：关于性能问题的概述。
  • 选择正确的连接：描述对站点最适宜的 Internet 连接类型的数学公式。
  • 计算连接性能：计算所需连接大小的计算器应用程序。
  • 限制带宽：改变 Web 服务器和单独的 Web 站点使用的带宽。
  • 限制连接数和设置连接超时：设置对 Web 服务器允许的连接数量的限制。
  • 启用保持 HTTP 激活：使用保持 HTTP 激活以维持打开的连接。
  • 启用 HTTP 压缩：使用压缩以减少网络流量。

本节不包括：

  • 有关创建事务和独立的应用程序的性能效果；请参阅配置应用程序。
  • 跟踪 Web 站点通讯；请参阅记录站点活动。
