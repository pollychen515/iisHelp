升级 IIS 群集资源

要点 本节描述的群集功能仅适用于 Windows 2000 Advanced Server。

IIS 支持升级和滚动升级群集服务资源。IIS 群集服务资源的升级要求在升级过程中所有的群集 IIS 资源处于脱机状态。相反，“滚动升级”允许客户在升级过程中使用群集 IIS 资源。

 

要完成 IIS 群集服务资源的滚动升级
  1. 将所有资源从节点 A 移到节点 B。
  2. 升级节点 A。
  3. 升级成功后，将所有资源从节点 B 移到节点 A。
  4. 升级节点 B。

要点 在尝试滚动升级之前，请查阅所有的关于群集资源的文档。

在标准升级期间（也就是非滚动升级），IIS 资源必须被删除、脱机或更改可能的所有者列表，这样资源仅可以在支持的节点上联机。
